Help and Documentation

The 10th Usability Heuristic and how its use in your web design...

Help and Documentation are important considerations for user experience. An intuitive website may still need to nudge a user's in the right direcion, should they need help. It is important for this information to be easily reachable from any point in your website. Users don't necessarily like to read, nor be told what to do, so brevity and ease are helpful. When applicable, anticipate when users will need help and ensure that you provide them with the relevant information. We will explore the application of types and their subtypes below.

Proactive Help

Proactive help's aim is to prevent issues before there is a problem. As such it usually familiarizes users with an interface. The three scenarios where this is applicable are new users at first launch of interface, novice users gaining proficiency or existing users encountering new/redesigned interfaces.
Within this category, there are two different types of proactive help which we will furth discuss:
Push Revelation and Pull Revelation.

Push Revelation

This first type of assistance isn't specifically relevant to a user's goals. The information being pushed, is outside of the current context and as such, more randomized. As with the green welcome message encountered above, it wasn't necessary for your immediate goal. Like other users who may not want be troubled reading, push revelations should be made to be easily dismissable. Ignoring to implement this in one's design may frustrate users who are already familiar with your interface.

Pull Revelation

In contrast, pull revelations are contextual to a user's task, providing relevant information.

Different implementation

It is better to favour pull revelations over push. As with all proactive help, keeping it short, timely and concise is advantageous. Using verb-oriented phrases, written from the user's perspective will be helpful in implementing this in your design.

Reactive Help

Reactive Help is provided after a user encounters a problem. This may answer questions, troubleshoot or provide good old documentation for the curious few. Documentation should be comprehensive and detailed, with no assumption made on behalf of the user. Visual aids are very helpful in providing information, both with tutorials/videos and material design. Providing effective search methods (ie grouping related help topics) make finding help easier and more efficient.

Great care should be taken to include help and documentation in the design of your website. A user's ability and familiarity varies, so providing help and documentation for a newcomer coming for the first time, or a pro for 1000th time means a certain balance must be struck. Whether providing non-contextual help on start...

...or providing documentation for those willing to read through it, help must always be accessible, unassuming, easy to scan and to the point.
